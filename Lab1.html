Conceptos

URL

Uniform Resource Locator, dirección de cualquier página visible en internet.

MIME

Multipurpose Internet Mail Extensions, estándar diseñado con el propósito de incrementar las capacidades de la mensajería electrónica y la variedad de archivos adjuntables.

Métodos y códigos de errores en HTTP

Errores de cliente/usuario:

400 Bad Request
Errores de sintaxis en la solicitud.
401 Unauthorized
Autenticación requerida para dar acceso.
403 Forbidden
El cliente no posee los permisos necesarios para dicho contenido.
404 Not Found
El servidor no puede encontrar el contenido solicitado.
408 Request Timeout
El servidor planea cerrar sesión por inactividad.

Errores de servidor:

500 Internal Server Error
El servidor no sabe qué hacer en esta situación.
501 Not Implemented
El método solicitado no está implementado por lo que el servidor se queda sin realizar la petición.
502 Bad Gateway
Se obtuvo una respuesta inválida mientras el servidor trabajaba como enlace.
503 Service Unavailable
El servidor no está listo para trabajar la petición (por sobrecarga o por mantenimiento)
504 Gateway Timeout
El servidor está actuando como una puerta de enlace y no puede obtener una respuesta a tiempo.
505 HTTP Version Not Supported
La versión de HTTP usada en la petición no está soportada por el servidor.

Metodos:

GET
Sólo debe recuperar datos, solicita una representación de un recurso específico.
HEAD
Parecido a GET, pero sin el cuerpo de la respuesta.
POST
POST envia una entidad a un recurso específico.
PUT
Reemplaza todas las representaciones actuales del destino.
DELETE
Borra un recurso en específico.
CONNECT
Genera un túnel hacia el servidor que haya sido identificado por el recurso.
OPTIONS
Se utiliza para describir las opciones de comunicación.
TRACE
Realiza una prueba de la ruta completa que va siguiendo el mensaje.
PATCH
Aplica modificaciones parciales a un recurso.

XML

Extensible Markup Language, diseñado para almacenamiento y transporte de datos. Las etiquetas son personalizadas.

XHTML

HTML extensible, HTML definido como XML, debe estar bien escrito y bien anidado

HTML5

La versión más reciente de HTML.

JS

JavaScript es un lenguaje multiparadigma, dicta el comportamiento de las páginas.

PHP

Hypertext Preprocessor, para páginas web dinámicas e interactivas, PHP trabaja del lado del servidor.

JSP

Java Server Pages, parecida a PHP y JS ya que genera páginas web dinámicas e interactivas, tiene acceso a todas las API’s de Java.

AJAX

Asynchronous Javascript and XML, permite a un servidor y a un navegador intercambiar información usando Javascript (tipo: XML, HTML, JSON) de forma asíncrona mediante programas escritos en Javascript.

Preguntas y respuestas

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es la conexión de dispositivos que siguiendo un mismo protocolo (TCP/IP) mientras que WWW es el servicio que permite el acceso a las páginas que comparten y crean los usuarios mediante la conexión a internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Control de datos; modificación, recuperación y eliminación de los mismos.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Post, le impide al cliente acceder a los datos en la URL (como cuando se usa get).

¿Qué método HTTP se utiliza cuando a traves de un navegador web se accede a una página a través de un URL?

Get, sólo carga la URL.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Se trata de una respuesta satisfactoria, quiere decir que la solicitud fue procesada con éxito y no hay error alguno.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No, es un error del servidor

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por que?

Sí, puede que haya una mala estructuración en una parte del código o que por los errores presentados el servidor no sepa cómo dar respuesta.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Algunas herramientas de HTML4 ya no son utilizadas en HTML5, también se busca que el código este más limpio y claro en HTML5

¿Qué componentes de estructura y estilo tiene una tabla?

Está compuesta de filas, columnas, bordes y colores.

Planificación, desarrollo, integración, mantenimiento, obsolescencia
